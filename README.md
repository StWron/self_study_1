C# 기반의 사전 입력된 데이터에서 원하는 요일의 날씨 정보를 추출하는 시스템

이 프로젝트는 학습 효율을 높이기 위하여 생성형 AI를 사용하였으며 논리적으로 더 효율적인 방식이 있는가에
탐구를 중점으로 작성하였음

개발 로그 (Development Log)

프로젝트 진행 중 발생한 문제점, 개선 과정, 그리고 이를 통해 학습한 내용을 기록

1. 데이터 입력의 구분

[초기 구현]
항목마다 배열을 생성하여 구현 현재는 항목 수가 6개 밖에 되지않아 문제가 없지만 개수가 늘어날수록 string의 숫자가
맞춰서 늘어난다는 생각을 하게됨

[문제 제기 및 개선 방향]
1차원 배열 여러개에서 2차원 배열 하나로 구성하여 출력시 new string[0,i],new string[1,i],new string[2,i],new string[3,i]
같은 방식으로 구현하여 for 문으로 1차원 배열 루프를 돌리는 방식에서 벗어남

2. 데이터 갱신시의 관리 방안

[초기 구현]
사전 데이터 입력하여 날씨를 조회하는 개념이니 만큼 데이터 추가 기능이 있어야 한다고 판단하여 새 날씨를 입력할경우 최신 날씨를 히스토리로 보내는 방식을 생각하였으며 맨 처음엔 shift기능을 생각하였으나 이럴 경우 데이터의 양이 많아질수록 연산해야 하는 분량이 늘어나지 않을까 하는 판단을 하여 고민을 한 결과 원형 버퍼(링 버퍼)의 개념에 접근하게됨

[문제 제기 및 개선 방향]
기존 접근 방식이 입력되는 데이터를 한칸 당기는(shift) 방식이라면 해당 방식은 i(보통 사용되는 변수)의 값 자체를 옮기고 사전에 입력된 데이터의 양은 숫자로 계산이 쉬우므로 마지막 값에 도달한 다음엔 0으로 초기화되는 방식을 채택

[추가 개선 가능성]
현재 첫번째 값이 검색하기위한 입력값과 같은데 해당 부분만 배열을 분리하여 그 부분내에서 검색을 하여 배열에서의 위치를 추출하여 i(변수)값으로 사용하여서 자동 입력하는 방식을 생각했으나 해당부분 데이터가 커질수록 검색에 필요한 자원이 커질것 같다는 판단을 함

[소감]
나름대로 고민해서 생각해낸 개념이 이미 다 있는 개념이라는 부분을 확인하면서 내가 사고하는 방식이 틀리지 않았다는 것을 확인하고 라이브러리와 알고리즘이란 개념에 대한 흥미를 가지게 됨 앞으로도 개인적으로 원하는 기능 구현을 위한 ai코딩(PRIVATE)와 연습(PUBLIC)을 병행할 예정
